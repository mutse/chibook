# 技术方案

## 1. 产品目标

复刻一款 Android/iOS 跨平台微信读书风格阅读 App，优先实现：

- 本地 EPUB/PDF 导入
- 书架与阅读进度管理
- 沉浸式阅读页
- AI 语音朗读
- 为后续的笔记、高亮、云同步、推荐系统预留架构

## 2. 技术选型

### 客户端

- Flutter
  - Android/iOS 单代码库
  - 阅读器类 UI 滚动性能稳定
  - 原生能力接入成熟，适合文件系统、TTS、SQLite

### 状态管理

- Riverpod
  - 解耦服务层、仓储层、界面层
  - 便于未来接入离线缓存和同步队列

### 路由

- GoRouter
  - 便于书架、详情页、阅读页、设置页的层级化管理

### 本地存储

- SQLite
  - 存储书籍元数据、导入路径、阅读进度、书签、笔记、朗读配置

### 文件导入

- `file_picker`
  - 支持选择 EPUB/PDF

### 阅读能力

- EPUB
  - 第一阶段先做解析接口和阅读容器
  - 建议第二阶段接入成熟 Flutter EPUB 渲染库或封装原生 WebView 渲染
- PDF
  - 第一阶段用成熟 PDF Viewer 组件落地

### 语音朗读

- 双通道方案
  - 本地 TTS：离线可用，启动快
  - 云端 AI TTS：音色更自然，可扩展章节级预生成缓存

## 3. 架构设计

分层结构：

- `app/`：应用初始化、主题、路由
- `features/bookshelf/`：书架、导入、最近阅读
- `features/reader/`：EPUB/PDF 阅读页、进度同步、朗读控制
- `features/settings/`：朗读偏好、接口配置
- `data/models/`：领域模型
- `data/repositories/`：仓储接口与实现
- `services/`：文件导入、解析、TTS、数据库、配置

核心数据流：

1. 用户选择本地文件
2. 导入服务复制文件到应用私有目录
3. 解析服务识别格式并提取基础元数据
4. 仓储层写入本地数据库
5. 书架展示书籍卡片
6. 阅读页根据格式分发到 PDF/EPUB 阅读器
7. 朗读服务按当前段落生成语音并驱动播放

## 4. 关键模块

### 4.1 导入模块

- 校验扩展名和文件大小
- 复制到应用沙盒目录，避免原文件被移动或删除后失效
- 生成封面、标题、作者等基础元信息
- 保存为 `Book` 记录

### 4.2 阅读模块

- PDF：页码驱动，进度按页存储
- EPUB：章节 + 段落偏移驱动，进度按 location/cfi 存储
- 统一抽象 `ReadingSession`，对上层屏蔽格式差异

### 4.3 AI 朗读模块

- `ReaderSpeechService` 统一暴露：
  - `speak(text)`
  - `pause()`
  - `resume()`
  - `stop()`
- 优先云端 AI TTS，失败时回退到本地 TTS
- 未来支持：
  - 章节缓存
  - 多音色
  - 倍速
  - 断点续播
